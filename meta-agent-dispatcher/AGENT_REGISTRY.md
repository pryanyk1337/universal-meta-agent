# РЕЕСТР АГЕНТОВ | База специализированных агентов

## СТРУКТУРА РЕЕСТРА

Каждый агент в реестре имеет:
- **ID**: Уникальный идентификатор
- **Имя**: Краткое название
- **Специализация**: Область экспертизы
- **Компетенции**: Конкретные навыки
- **Входы**: Что принимает на вход
- **Выходы**: Что производит
- **Зависимости**: Связи с другими агентами

---

## ДОМЕН: ЛИТЕРАТУРСТВО

### AGENT-001: Аналитик-Архивариус
**Специализация:** Анализ и структурирование литературного материала

**Компетенции:**
- Анализ текстов (стиль, жанр, темы)
- Систематизация персонажей, сюжетов, миров
- Создание баз знаний и справочников
- Выявление паттернов и мотивов

**Входы:**
- Литературные тексты (фрагменты, произведения)
- Концепции и идеи
- Исходные материалы проекта

**Выходы:**
- Структурированные базы данных персонажей
- Аналитические карты миров
- Тематические каталоги
- Справочники по миру произведения

**Связи:**
- → AGENT-002 (передаёт структурированные данные)
- → AGENT-003 (предоставляет справочные материалы)

---

### AGENT-002: Архитектор Сюжетов и Сценариев
**Специализация:** Проектирование сюжетных структур

**Компетенции:**
- Создание сюжетных арок
- Драматургическая структура (3-акт, Hero's Journey)
- Планирование конфликтов и поворотов
- Разработка персонажных дуг

**Входы:**
- Базы данных от AGENT-001
- Концепции сюжетов
- Требования к жанру и структуре

**Выходы:**
- Детальные сюжетные планы
- Сценарные схемы (по актам/главам)
- Таблицы событий и взаимосвязей
- Карты развития персонажей

**Связи:**
- ← AGENT-001 (получает аналитику)
- → AGENT-003 (передаёт сценарные планы)

---

### AGENT-003: Писатель
**Специализация:** Создание литературного текста

**Компетенции:**
- Художественное письмо (проза, диалоги, описания)
- Стилистическая адаптация
- Работа с голосом повествования
- Редактура и шлифовка текста

**Входы:**
- Сценарные планы от AGENT-002
- Справочные материалы от AGENT-001
- Стилистические требования

**Выходы:**
- Готовые литературные тексты
- Главы, сцены, эпизоды
- Диалоги и описания
- Финальные произведения

**Связи:**
- ← AGENT-001 (справочные данные)
- ← AGENT-002 (сценарные планы)

## ПРИНЦИПЫ РЕЕСТРА

### 1. Модульность
Каждый агент = независимый модуль с чёткими границами

### 2. Композируемость
Агенты комбинируются в цепочки для решения сложных задач

### 3. Специализация
Один агент = одна узкая область экспертизы

### 4. Расширяемость
Реестр постоянно пополняется новыми агентами

---

## ИСПОЛЬЗОВАНИЕ РЕЕСТРА

**Для мета-агента:**
1. Анализируй задачу пользователя
2. Ищи подходящих агентов в реестре
3. Составляй цепочку выполнения
4. Координируй передачу данных между агентами

**Для пользователя:**
1. Изучай доступных агентов
2. Выбирай подходящего под задачу
3. Следуй инструкциям агента
4. Комбинируй агентов при необходимости
