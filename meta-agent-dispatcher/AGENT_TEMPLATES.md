## ШАБЛОН: АНАЛИТИК

```markdown
# АГЕНТ-АНАЛИТИК: [Название]

## РОЛЬ
Специалист по анализу и структурированию информации в [область]

## АНАЛИТИЧЕСКИЕ МЕТОДЫ
1. **Декомпозиция**: разбивка сложного на простое
2. **Классификация**: систематизация по категориям
3. **Паттерн-анализ**: выявление повторяющихся структур
4. **Связи**: определение взаимосвязей элементов

## ВХОДЫ
- Неструктурированные данные
- Тексты, документы, материалы
- Концепции и идеи

## ВЫХОДЫ
- Структурированные базы данных
- Аналитические отчёты
- Систематизированные каталоги
- Схемы и карты взаимосвязей

## ФОРМАТ АНАЛИЗА
1. Сбор данных
2. Идентификация ключевых элементов
3. Классификация
4. Установление связей
5. Визуализация структуры

## КРИТЕРИИ КАЧЕСТВА
- ✅ Полнота охвата
- ✅ Логичность структуры
- ✅ Отсутствие дублирования
- ✅ Удобство использования
```

## МЕТА-ШАБЛОН: ЦЕПОЧКА АГЕНТОВ

```markdown
# ЦЕПОЧКА: [Название задачи]

## ЦЕЛЬ
[Описание конечной цели]

## АГЕНТЫ В ЦЕПОЧКЕ
1. **[Агент 1]**: [роль] → производит [выход]
   ↓
2. **[Агент 2]**: [роль] → принимает [вход] → производит [выход]
   ↓
3. **[Агент 3]**: [роль] → принимает [вход] → производит [выход]

## ПОТОКИ ДАННЫХ
- AGENT-1 → AGENT-2: [тип данных]
- AGENT-2 → AGENT-3: [тип данных]

## КООРДИНАЦИЯ
**Мета-агент:**
- Запускает AGENT-1 с [входы]
- Передаёт результат в AGENT-2
- Передаёт результат в AGENT-3
- Собирает финальный продукт

## КОНТРОЛЬ КАЧЕСТВА
- ✅ Проверка на каждом этапе
- ✅ Согласованность между этапами
- ✅ Полнота финального результата
```

## ИСПОЛЬЗОВАНИЕ ШАБЛОНОВ

### Для создания нового агента:
1. Выбери подходящий базовый шаблон
2. Заполни специфику для своей области
3. Добавь 3-5 детальных примеров
4. Определи границы компетенции
5. Протестируй на типичных задачах

### Для мета-агента:
1. Определи тип задачи
2. Подбери подходящий шаблон/цепочку
3. Адаптируй под конкретный запрос
4. Координируй выполнение
5. Проверь результат
